## ūüĒĄ Data Flow

How a single user prompt traverses the system:

```mermaid
sequenceDiagram
    autonumber
    participant U as ūüĎ§ User
    participant UI as ūüĖ• UI (CLI/Electron/Web/VSCode)
    participant ROUTER as ūüďč CORTEX.md Router
    participant REPL as ūüĒĀ Agentic REPL
    participant TOOLS as ūüĒß Tools
    participant MCP as ūüĒĆ MCP Servers
    participant LLM as ūüß† HF Router (GLM-5)
    participant FALLBACK as ūüĒÄ Fallback (novita / Ollama)
    participant FS as ūüíĺ Filesystem / DB

    U->>UI: "create portfolio site at ~/Desktop/..."
    UI->>ROUTER: raw prompt
    ROUTER->>ROUTER: classify ‚Üí match playbook from CORTEX.md
    ROUTER->>REPL: enriched prompt + system context

    loop Agentic Loop (observe ‚Üí think ‚Üí act)
        REPL->>LLM: POST /v1/chat/completions
        alt Primary OK
            LLM-->>REPL: tool_calls + reasoning
        else 5xx / empty
            LLM->>FALLBACK: retry with :novita
            FALLBACK-->>REPL: tool_calls + reasoning
        end

        alt Tool: Read/Write/Edit/Bash
            REPL->>TOOLS: invoke
            TOOLS->>FS: fs.readFile / writeFile / exec
            FS-->>TOOLS: result
            TOOLS-->>REPL: observation
        else Tool: MCP
            REPL->>MCP: jsonrpc tool call
            MCP-->>REPL: result
        else Done
            REPL->>UI: final answer / file paths
        end
    end

    UI->>U: streamed response + artefacts
```

### ūü™ú Multi-tier Fallback Chain

Cortex automatically falls back when the primary provider returns
`401`, `403`, `429`, or `5xx`. The chain is configurable end-to-end:

```
1. HF primary          (HF_MODEL_ID)
   ‚Üď fails
2. HF fallback list    (HF_MODEL_FALLBACK ‚ÄĒ comma-separated, tried in order)
   ‚Üď all fail
3. NVIDIA              (NVIDIA_API_KEY + NVIDIA_MODEL_ID)
   ‚Üď fails
4. Groq                (GROQ_API_KEY + CORTEX_GROQ_FALLBACK_MODEL)
   ‚Üď fails
5. Ollama (local)      (CORTEX_OLLAMA_FALLBACK_MODEL ‚ÄĒ works offline)
```

Every step prints a colored status banner to the terminal so you always see
which provider is serving the request:

```
‚úó HF primary failed   ‚Ēā moonshotai/Kimi-K2.6:together  (HTTP 403)
‚ÜĽ trying fallback     ‚Ēā HF ‚Üí zai-org/GLM-5:together
‚úď using HF fallback   ‚Ēā zai-org/GLM-5:together
```

The active model also appears in the spinner next to the verb:
`Twisting‚Ä¶ ¬∑ Kimi-K2.6`.

### ūü§Ě Shared Sessions ‚ÄĒ `/share` (multiplayer Cortex)

‚ÄĒ like Google Docs for a terminal coding session.

```

   ‚Üď
```

Turn it on with one env var: `CORTEX_DUAL_MODEL=1`. Each chat-completion request
is auto-routed: if the agent loop is calling a tool ‚Üí Executor; if it's
producing a plan/reflection ‚Üí Planner. Falls back to the single-model path when
the env var is unset, so existing setups keep working.

### ūüíď Emotion Layer (tone-aware responses)

Cortex can detect your emotional state from each message and adapt its tone
like a human would ‚ÄĒ empathetic when you're stuck, concise when you're in a
hurry, warm when you're happy.

```bash
CORTEX_EMOTION_LAYER=1     # local heuristic (zero cost, ~0.1ms/turn)
# or
CORTEX_EMOTION_LAYER=hf    # HuggingFace classifier (adds ~200ms/turn)
CORTEX_EMOTION_DEBUG=1     # print detected emotion each turn (stderr)
```

**Detects 8 emotions:** `frustrated` ¬∑ `angry` ¬∑ `confused` ¬∑ `urgent` ¬∑
`happy` ¬∑ `curious` ¬∑ `sad` ¬∑ `anxious`.

**How it works:** before every turn, `@src/services/emotion/detector.ts` scans
the latest user message for signals (profanity, ALL-CAPS, `!!`, "doesn't
work", "urgent", "thanks", "how do I", ‚Ä¶). When confidence ‚Č• 0.25 a
one-line tone hint is prepended to the system prompt, e.g.:

> *"User is frustrated. Be empathetic in one short line ('got it ‚ÄĒ fixing
> now'), then solve directly. No lectures, no filler, no long preambles."*

Verified on 10 test cases ‚ÄĒ see the regression table in
`@src/services/emotion/detector.ts`.

### Adding a new slash command

All Tier 2-9 commands share a factory in `src/commands/_tierHelper.ts`:

```ts
import { makeTierCommand, CODE_EDIT_TOOLS } from '../_tierHelper.js'

export default makeTierCommand({
  name: 'my-command',
  aliases: ['mc'],
  description: 'Does X',
  progressMessage: 'doing X',
  allowedTools: CODE_EDIT_TOOLS,
  buildPrompt: (args) => `## Protocol\n‚Ä¶your prompt here‚Ä¶`,
})
```

Register it in `src/commands.ts` ‚Üí import + add to the `COMMANDS()` array ‚Üí `bun run build`.

### Adding a new agent

Drop a `.md` or `.yaml` file into `src/skills/agency/<category>-<name>.md`. It's auto-discovered at startup ‚ÄĒ no registration needed.

### Adding a new MCP server

Edit `.mcp.json`:

```json
"my-server": {
  "command": "npx",
  "args": ["-y", "@example/mcp-server"],
  "env": { "MY_TOKEN": "${MY_TOKEN}" }
}
```
